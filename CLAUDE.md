# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

Playwright-based end-to-end test suite for the Procertif certification platform (`https://app.procertif.dev`). Tests are written in TypeScript and cover certification evaluation flows (quizzes) and jury review workflows. The UI and tests are in French.

## Commands

```bash
# Run all tests (Playwright CLI)
npm test

# Run a specific test file
npx playwright test cas1-quiz.spec.ts

# Start the web-based test runner UI (port 3333)
npm run server

# Docker
docker compose up
```

No linting or build step is configured.

## Environment

Configuration lives in `.env` (not committed):
- `BASE_URL` — target app URL (default `https://app.procertif.dev`)
- `TEST_OTP` — OTP code for magic-code login (default `444444`)
- `PORT` — server port (default `3333`, requires restart)
- `HEADLESS` — set to `false` to run Chromium in headed mode (default `true`)
- `LANG` — UI language: `fr` or `en` (default `en`)
- `ANTHROPIC_CLIENT_ID` — OAuth client ID for Claude API
- `ANTHROPIC_MODEL` — Claude model used by the chat (e.g. `claude-sonnet-4-6`)

## Architecture

### Test Files

Test specs live in `tests/` (gitignored — generated by the chat IA or created manually), named with the pattern `<domain>_<type>.spec.ts` (e.g. `badge_competences_ia.spec.ts`, `titre_rncp.spec.ts`). The list of specs varies per environment and is not tracked in the repository.

All tests follow the same structure: set up a `screenshots/<test-name>/` directory → navigate directly with a `VALIDATION_TOKEN` in the URL → interact with UI → capture screenshots at key steps. Tests contain no assertions — they focus on workflow capture. The validation token is hardcoded in each test file and passed as `?validation=<token>`. Authentication logic is inlined in each test file (no shared helpers).

### Web Test Runner (`app/`)

A Node.js HTTP server (`app/server.js`) that provides a full UI for running and managing tests:

**Test execution**
- `GET /api/tests` — list test files with metadata (cas number, type, estimated duration)
- `POST /api/run/:filename` — spawn Playwright as a child process
- `POST /api/kill/:runId` — kill a running test process (SIGKILL)
- `GET /api/run-status/:runId` — get run status (`running` / `passed` / `failed`)
- `GET /api/stream/:runId` — stream output via Server-Sent Events
- `POST /api/session` / `GET /api/session` — save/load last session data

**Test management**
- `GET /api/test-aliases` — list all display-name aliases
- `PUT /api/test-aliases/:testkey` — set or clear an alias (body: `{alias}`)
- `DELETE /api/test-aliases/:testkey` — remove alias
- `DELETE /api/tests/:testkey` — delete test file + all associated data (screenshots, actions, spec, aliases, groups, history)

**Screenshots**
- `GET /api/screenshots` — list screenshot groups
- `GET /api/screenshots/:folder` — get screenshot count for a folder
- `GET /screenshots-img/:folder/:file` — serve individual screenshots
- `DELETE /api/screenshots/:folder` — delete a screenshot folder

**Groups**
- `GET /api/groups` — list groups (persisted in `data/groups.json`)
- `POST /api/groups` — create group
- `PUT /api/groups/:id` — update group
- `DELETE /api/groups/:id` — delete group

**Spec generation**
- `POST /api/spec-regen/:testname` — regenerate a French Gherkin spec from test code + action log
- `GET /api/spec/:testname` — get generated spec (Markdown)
- `GET /api/actions/:testname` — get recorded action list (JSON)
- `GET /api/prompt/:testname` — get saved prompt history for a test (JSON)

**Pending test modifications**
- `GET /api/pending` — list tests with pending modifications (generated by chat, not yet confirmed)
- `POST /api/pending/:testname/run` — run a pending test in isolation
- `POST /api/pending/:testname/confirm` — promote pending test to `tests/`
- `POST /api/pending/:testname/discard` — discard pending modifications

**Claude chat**
- `POST /api/chat` — start a chat turn (message, images, sessionId, instructions)
- `GET /api/chat-stream/:runId` — stream chat events (SSE)
- `POST /api/chat-stop/:runId` — interrupt a running chat generation
- `POST /api/chat-save` — save conversation as JSON to `tests/prompt/<name>.json`
- `GET /api/chat-logs` — list all chat log files with token usage summaries
- `GET /api/chat-logs/:filename` — full detail of a chat log (API calls + messages)
- `DELETE /api/chat-logs/:filename` — delete a chat log file

**Configuration**
- `GET /api/config` — read current `.env` as JSON
- `POST /api/config` — write `.env` from JSON body
- `GET /api/lang` — return current language (`fr` or `en`)
- `GET /i18n/:lang.json` — serve translation file

UI pages served from `app/`:
- `index.html` — Main test runner (queue, real-time output, session tracking)
- `screenshots.html / screenshots.css` — Screenshot viewer with lightbox
- `groups.html / groups.css` — Test group creation and management
- `chat.html / chat.css` — Claude chat interface with tool execution, global instructions, and conversation save
- `scenarios.html / scenarios.css` — Scenarios viewer (Gherkin specs per test) and pending workflow
- `logs.html / logs.css` — Chat session history with token usage and API call details
- `config.html` — `.env` editor (known vars + custom vars)
- `i18n/en.json`, `i18n/fr.json` — UI translations
- `i18n.js` — Client-side i18n loader (loaded before page scripts via `<script src="/i18n.js">`)

### Data Files (`data/`)

Runtime data, gitignored:
- `groups.json` — Test grouping configuration
- `last-session.json` — Results of the last test session
- `run-history.json` — Historical timing metrics per test file
- `test-aliases.json` — Display name aliases for test files
- `actionTest/` — Per-test JSON records of recorded UI actions
- `specs/` — Generated Gherkin specs (Markdown) per test
- `pending/` — Pending test specs generated by chat IA (awaiting confirmation)
- `promptTest/` — Per-test prompt histories
- `chat-logs/` — Chat session logs (JSON, one file per session)

Note: run logs (`run-<testname>-last.log`, `run-<testname>-last-failed.log`) are no longer written to disk.

### Playwright Config (`playwright.config.ts`)

- Base URL: `process.env.BASE_URL || "https://app.procertif.dev"`
- Browser: Chromium only
- `headless: process.env.HEADLESS !== "false"`
- `fullyParallel: false` — tests run sequentially
- `retries: 0`
- `trace: "off"`, `screenshot: "off"` — disabled by default
- `ignoreHTTPSErrors: true`
- `testDir: "tests"` — all `*.spec.ts` files in that directory are executed

### Docker

- `Dockerfile` — based on `node:20-bookworm`, installs deps + Playwright Chromium, exposes port 3333
- `docker-compose.yml` — host network, X11 mount for headful display, volume mounts for `app/`, `tests/`, `data/`, `screenshots/`, and host `.claude` credentials

## Key Patterns

**Authentication**: Tests bypass the login screen by passing a `VALIDATION_TOKEN` as a URL query parameter (`?validation=<token>`). The token is hardcoded in each test file. No email or OTP interaction is required.

**Timeouts**: Tests set `test.setTimeout(120000–240000)`. Element waits use explicit timeouts (10s–60s). After navigation, wait for network idle and for animations to finish:
```ts
await page.waitForFunction(() =>
  document.getAnimations().every(a => a.playState !== "running")
);
```

**Selectors**: Prefer role-based locators (`page.getByRole("button", { name: "..." })`); fall back to CSS selectors (`#otp_token`, `.form-check-input`) when needed.

**Screenshots**: Captured between every action via a numbered helper defined at the top of each test:
```ts
const shot = (p: Page, n: number, label: string) =>
  p.screenshot({ path: path.join(SCREENSHOTS_DIR, `${String(n).padStart(2, "0")}-${label}.png`), fullPage: true });
```
This convention (numbered + labelled PNGs) is enforced as a global instruction in the chat IA.

**Chat IA system prompt**: The chat embeds two fixed system blocks (identity + working dirs) plus an optional user-defined instructions block stored in `localStorage`. Default instruction: `"Toujours prendre un screenshot entre chaque action dans les tests."` The chat can execute tools (Read, Write, Edit, Bash, Glob, LS, WebFetch, ReadImage) to generate and modify test files. Conversations can be saved as JSON to `tests/prompt/` via the save modal. The system prompt tells the model that the Procertif source code is at `/app/webapp/` and the e2e suite is at the root of this repo.

**Chat interleaved tool display**: Tool-use pills are rendered inline in the assistant bubble at the exact position they were called, not all grouped at the top. The bubble is built dynamically: `delta` events append to the current text element; `tool_start` events finalize the current text segment, insert a `.tool-pills` container at the current position, then reset the text state. Consecutive tool calls share the same `.tool-pills` container. Each assistant message saves a `blocks` array (`[{type:'text',text}, {type:'tool',name,input}]`) alongside `content` and `tools` to allow faithful restore from localStorage.

**Pending workflow**: Tests generated by the chat IA land in a pending state and must be confirmed via the `/scenarios` UI or the API before being moved to `tests/`.

**i18n**: All UI strings go through `window.t(key)`. The `i18n.js` script fetches the current language from `/api/lang`, loads the corresponding translation file, populates `window.t`, and fires an `i18nLoaded` DOM event. Pages listen for `i18nLoaded` to run their init code. Language is controlled by the `LANG` env var and can be changed from the `/config` page (triggers a reload).
